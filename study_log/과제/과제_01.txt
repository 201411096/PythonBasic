--------------------문제--------------------
1. 홈디렉토리와 루트디렉토리
2. su root 와 su - root 차이점
3. rmdir  / rm
4. 마운트
----------------------------------------
1.
ㄴ 루트디렉토리 : 모든 디텍토리의 최상위 디렉토리
ㄴ 홈디렉토리 : 사용자의 홈 디렉토리, useradd 명령어로 새로운 사용자 생성시 사용자의 동일한 이름의 디렉토리가 자동으로 생성됨
2.
ㄴ su root   : su는 root 계정의 환경변수를 가져오지 않고, 현재 계정의 환경 변수를 그대로 사용함
ㄴ su - root : su - 로그인시 환경변수를 가져오면서 /root 디렉토리로 이동
3. 
ㄴ rmdir : 디렉토리를 삭제할 때 사용하는 명령어
	ㄴ 비어있는 디렉토리만을 제거할 수 있음
ㄴ rm : 파일 혹은 디렉토리를 삭제할 때 사용하는 명령어
4. 
ㄴ 마운트 : 보조기억장치(HDD, FDD, CD-ROM)나 파일 시스템이 다른 디스크를 /의 하위 디렉토리로 연결하여 사용 가능하게 해주는 명령어

--------------------------------------------------------------------------------
1. 홈디렉토리와 루트디렉토리 
	/(루트) 
	가장 최상위 디렉토리, 
	리눅스의 모든 디렉토리들의 시작점이다. 
	(즉, 모든 디렉토리들을 절대경로로 표기할 때에 이 디렉토리로부터 시작해야 한다.)
	
	/home
	시스템 계정 사용자들의 홈 디렉토리 
	useradd 명령어로 새로운 사용자를 생성하면 대부분 사용자의 ID와 동일한 이름의 디렉토리가 자동으로 생성됨.

2. su root 와 su -root의 차이점
	su root
	전환 하려는 계정의 환경변수는 가져오지 않는다.
	디렉토리 유지

	su -root
	전환하려는 계정의 환경변수를 가져올 수 있다.
	홈디렉토리로 감

3. rmdir  / rm
	rmdir 
	remove directory의 약자로 디렉토리를 지우는 명령어
	단, 지우고하는 디렉토리안이 비어있지 않으면 지울 수 가 없다.
	rmdir [option] directory(s)
	
	rm
	remove의 약자로 파일을 지우는 명령으로 옵션에 다라 디렉토리도 지운다.
	rm -i: 현재 디렉토리의 모든 파일을 하나씩 확인하면서 지운다.
	rm -rf tmp: tmp라는 디렉토리와 그 안의 모든 파일들을 허용권에 관계없이 강제요 지운다.
	
4. 마운트
	보조기억장치(HDD, FDD, CD-ROM 등)나 파일 시스템이 다른 디스크를 
	/의 하위 디렉토리로 연결하여 사용 가능하게 해주는 명령어

	사용법
	mount[option] [device] [directory]	
		옵션
		-a : /etc/fstab에 명시된 파일 시스템을 마운트할 때 쓰이는 옵션입니다.
		-t fs_type: 파일 시스템의 유현을 지정하는 옵션으로 지정하지 않으면, /etc/fstab 파일 참조
		-o 항목: 마운트할 때 추가적인 설정을 적용할 때 사용하는 옵션으로 다수의 조건을 적용할 때는 콤마(,)로 구분하면 됩니다.
			ro: 읽기 전용으로 마운트
			rw: 읽기 쓰기 모드(default 값)
			remount: 해당 파티션을 다시 마운트(파티션 정보가 바뀌었을 때 사용)
			loop: loop 디바이스로 마운트할 때 사용(CD-ROM 이미지 파일인 iso를 마운트할 때 사용)
			noatime: 파일의 내용을 읽게 되면 access time이 변경되는데, 이 옵션을 사용하면 파일이 변경되기 전까지는 Access time이 변경 되지 않음
				이에따라 시스템 작업을 줄여 성능 향상(노트북은 베터리 시간 증가(
			
		디스크 추가 삭제하는 법
		1. 마운트할 장소 만들기
		mkdir /<마운트 추가할 장소>
		2. 디스크 리스트를 불러온다.
		fdisk -l
		(fdisk: 디스크 파티션을 확인하고 추가/삭제하는 명령)
			-l: 현재 디스크의 파티션 테이블 정보 출력
			-s partition: 특정 파티션의 크기를 출력(단위: block)
		3. 디스크에 대한 세부 값을 읽어옴 
		blkid
		4. 특정 디스크 마운트
		mount /<디스크> /<마운트 추가할 장소>
		5. 특정 디스크 마운트 해제
		unmount /<디스크>
